# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public issue.**

### Contact

Send an email to: **security@example.com** *(à remplacer par une vraie adresse)*

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment**: within 72 hours
- **Assessment**: within 7 days
- **Fix**: best effort, typically within 30 days
- **Disclosure**: 90 days after initial report

### Scope

- Web application (frontend + API)
- Infrastructure and deployment configuration
- Dependencies

### Out of scope

- Social engineering attacks
- Denial of service attacks
- Issues in third-party services (twitterapi.io, HuggingFace)

## Supported versions

Only the latest version on the `main` branch is supported.
